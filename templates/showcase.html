Install and setup

Basic commands for getting EmilyCSS running in a project.

terminal 
npm install emily-css
terminal 
npx emily-css init
terminal 
npm run emily:watch
terminal 
npm run emily:build
index.html 
<link rel="stylesheet" href="./dist/emily.min.css">

Build output

How EmilyCSS handles local development and production output.

dist/emily.min.css

Production-ready CSS. This is the file you link to in your project.

dist/emily.css

Temporary local framework source. Watch keeps it available when needed.

emily-css build

Outputs dist/emily.min.css and removes dist/emily.css by default.

Development

Run npm run emily:watch. It keeps a full local emily.css available when needed and regenerates emily.min.css as classes change.

Production

Run npm run emily:build. Production should only ship dist/emily.min.css.

Pattern components

Single classes that combine multiple utilities into named patterns. They live in @layer components so utility classes always override them.

.prose

Inside .prose. Line length is capped at 65 characters and the block centres itself — ideal for article copy and documentation.

prose.html 
<div class="prose">
  <p>Article copy here.</p>
</div>
<!-- max-width: 65ch; margin-inline: auto -->
